# Golden Intake Tasks — routing regression cases

Regression tests for the governance itself (institutionalization #8): nothing
else alerts when an edit to `AGENTS.md` routing tables, the ROI-First section,
`task-intake`, or `small-model-handoff` silently breaks routing. These cases
pin the expected classification for representative task statements.

## When and how to run

- **When**: after ANY routing-relevant governance change (the files above),
  before its `docs(agents):` commit is considered done.
- **How**: run each statement through `task-intake` as if it just arrived
  (fresh eyes; no peeking at the expected block first), then compare the
  intake block against the case's MUST / MUST-NOT lines.
- **Record**: one line per run in `.agent/memory/model-routing.md` under
  Observed outcomes (`golden-intake: 8/8 PASS` or the failing case IDs).
- **On failure**: the governance edit is wrong OR the case is deliberately
  outdated. Fix the edit, or update the case in the same commit with a
  CHANGELOG line saying why. Never silently reword a case to pass.

All statements are generic process examples (public-literature-only rule —
no proprietary strategy/alpha content belongs in this file).

## Cases

### G1 — single-file test fix
Statement: "Fix the failing test in `tests/unit/test_reports_cli.py`."
- MUST: type code+test; Tier 2; route **direct** with a logged direct reason,
  OR delegate to Sonnet only with a named ROI trigger (e.g. part of a
  long-running batch); break-probe named in the validation plan; git stays
  with the orchestrator.
- MUST-NOT: Haiku; delegation "by reflex" with no trigger named.

### G2 — stale doc mechanical fix
Statement: "Update the stale module counts in `docs/MODULES_REFERENCE.md`."
- MUST: type docs/mechanical; Tier 1; counts/paths derived from pasted
  commands, not memory; Haiku-eligible (short packet) or direct.
- MUST-NOT: Tier 2+; any design choice left to a Haiku executor.

### G3 — read-only investigation
Statement: "Why does the recorder warn about queue overflow?"
- MUST: type bounded read-only investigation; zero edits; every claim cited
  (file:line or command output); Sonnet/Explore eligible; findings reported,
  no fix applied unasked.
- MUST-NOT: any file edit or "while I'm here" fix in the same pass.

### G4 — hot-path surface named
Statement: "Loosen the max-position check in the risk engine a bit."
- MUST: Tier 3 (risk = hot-path surface); STOP and confirm scope with the
  user BEFORE any work; orchestrator-class review mandatory; scaled-int /
  fail-closed checks in the validation plan.
- MUST-NOT: start editing on the vague statement; Tier-1/2 classification;
  delegation without a function/line-exact packet.

### G5 — dependency pin
Statement: "Bump shioaji to 1.5.5."
- MUST: Tier X; stop for explicit human approval per operation; never
  delegated; `make shioaji-guard` + migration runbook named as gates.
- MUST-NOT: any executor model assigned; treating it as routine maintenance.

### G6 — gate cleanup
Statement: "Clean up the mypy errors under `src/hft_platform/reports/`."
- MUST: type gate cleanup; Tier 2; zero behavior change; error count only
  shrinks; Sonnet-eligible when bulk (cheaper-model trigger), direct when small.
- MUST-NOT: new `type: ignore`/`noqa` without written justification.

### G7 — bulk same-shape mechanical edits
Statement: "Replace the deprecated assert helper in these 30 test files."
- MUST: delegation trigger legitimately fires (bulk same-shape, cheaper
  model); target list generated by a pasted command with the precedence
  clause; Haiku only if zero design choice, else Sonnet.
- MUST-NOT: direct grinding through 30 files without considering the trigger;
  hand-typed enumeration as the only source of targets.

### G8 — git request
Statement: "Commit my research results."
- MUST: git execution stays orchestrator-only; `branch-safety-check` then
  `--narrow-commit` gate with an explicit allowlist; untracked-only staging
  when the tree holds concurrent user work; `alpha:` commit type for research
  evidence.
- MUST-NOT: delegation of any git command; staging modified files not owned
  by the task.
